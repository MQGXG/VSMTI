# Mira 下个版本改进计划

## 背景

当前版本已完成 6 项低风险改进（快照持久化、FTS5 搜索、Structured Summary、Dream 知识注入、Event Sourcing、Todo 系统）。下个版本聚焦架构级重构。

---

## 4 项改进总览

| 序号 | 改进项 | 当前行数 | 目标行数 | 工作量 |
|------|--------|---------|---------|--------|
| 1 | ToolRegistry 拆分 | 340行 | ~70行（核心） | 2d |
| 2 | Agent.run() 瘦身 | 738行 | ~500行 | 2d |
| 3 | 权限 Deferred 增强 | 460行 | 460行（优化） | 1d |

**总计：约 7 天**

## 第 3 步：权限 Deferred 增强（1 天）

### 现状

权限系统**已经实现了 Deferred 模式**：
- `stateMachine.createPermissionRequest()` 创建 Promise
- `yield permission_request` 事件给 UI
- `await waitForReply()` 挂起等待
- `replyPermission()` 解析 Promise

### 增强方向

不需要大改，只需优化体验：

**1. 批量权限审批**
- 当 Agent 一次发出多个相同类型的写操作时，合并为一次审批
- 在 `turn-runner.ts` 的 `quickPermissionCheck` 中检测批量场景

**2. 权限规则可视化**
- 在 UI 中展示当前生效的权限规则链
- 新增 `getPermissionRules()` API 返回规则列表

**3. 权限规则导入/导出**
- 支持 `.mira/permissions.json` 文件
- 新增 `importPermissions()` / `exportPermissions()` API

### 修改文件

| 文件 | 变更 |
|------|------|
| `system/permission/index.ts` | 新增 `getPermissionRules()` |
| `system/permission/store.ts` | 新增 `importPermissions()` / `exportPermissions()` |
| `agent/turn-runner.ts` | 批量权限审批逻辑 |

- Provider Failover Chain
- 成本/延迟自动切换
- Model→Provider 自动映射

### 新文件：`packages/core/src/llm/provider-policy.ts`

**职责**：Provider 策略引擎

```typescript
export interface ProviderRule {
  effect: "allow" | "deny"
  action: string       // "use" | "route" | "fallback"
  resource: string     // provider name 或 model name（支持通配符）
}

export interface ProviderPolicy {
  rules: ProviderRule[]
}

export class ProviderPolicyEngine {
  private rules: ProviderRule[] = []

  // 加载策略
  load(rules: ProviderRule[]): void

  // 评估是否允许使用某个 Provider
  evaluate(providerName: string, modelName?: string): "allow" | "deny"

  // 从配置文件加载策略
  loadFromFile(filePath: string): void
}
```
